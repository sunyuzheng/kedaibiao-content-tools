# 核心任务说明

> 本项目核心目标：本地有完整、准确的视频资料，Transistor 上有正确顺序的播客。

---

## 标准工作流

```
1. 下载    → ./tools/download/download_channel.sh
             （自动跳过会员视频；下载后自动运行 organize）
2. 对账    → python3 tools/check/check_upload_candidates.py
             （查看有多少新的有人工字幕视频需要上传）
3. 上传    → python3 tools/upload/upload_to_transistor_v2.py --upload-only-new --dry-run
             （先 dry-run 确认 EP 号和标题，再去掉 --dry-run 正式上传）
4. 质检    → python3 tools/check/check_upload_quality.py --n 5
             （上传后必跑：验证 published_at、image_url、video_url、标题格式等）
5. 排序    → python3 tools/upload/reorder_episodes_by_date.py
             （首次或有新 EP 时运行，确保 EP 号与 YouTube 发布时间一致）
```

---

## 各步骤说明

### 1. 下载

```bash
./tools/download/download_channel.sh
```

- 需要 yt-dlp 最新版：`brew upgrade yt-dlp`（版本过旧会导致新视频静默失败）
- 从 Chrome 读取 Cookie（`--cookies-from-browser chrome`）
- 自动跳过会员专属视频（`availability != subscriber_only & availability != needs_auth`）
- 下载完成后自动运行 `organize_subtitles.py` 分类

**下载内容**：音频（m4a）、info.json、description、字幕（srt）、缩略图（webp）

**分类规则**（`tools/organize/organize_subtitles.py`）：
- info.json `subtitles` 字段有 zh/en → `archive/有人工字幕/`
- 其他 → `archive/无人工字幕/`

> ⚠️ 有 `.srt` 文件不等于有人工字幕。判断依据是 info.json 的 `subtitles` 字段，不是文件存在。

### 2. 对账

```bash
python3 tools/check/check_upload_candidates.py
```

直接打印结果，无文件输出。

**关注**：`有人工字幕 未上传` 数量 → 这些需要上传

> 日常用 `check_upload_candidates.py`（只看比最新已发布更新的视频）；
> 历史全量审计用 `check_transistor_sync.py`（扫描所有本地视频，含历史漏网）。

### 3. 上传

先 dry-run 确认候选列表和 EP 号：
```bash
python3 tools/upload/upload_to_transistor_v2.py --upload-only-new --dry-run
```

确认无误后上传：
```bash
python3 tools/upload/upload_to_transistor_v2.py --upload-only-new
```

**上传逻辑**：
- 只上传比「已发布最新内容」更新的视频（按 YouTube 发布日期过滤，防止历史漏网视频混入）
- 按日期升序上传，旧的先传 → 新视频拿到最大 EP 号
- 上传内容：音频 + 标题（`E{N}. 标题` 格式）+ description + YouTube video_url + image_url（hqdefault.jpg）
- **published_at 通过 `/publish` 端点设置**（不能用 PATCH /episodes/:id，会被忽略）

### 4. 质检（上传后必跑）

```bash
python3 tools/check/check_upload_quality.py --n 5
```

检查最新 5 个 episodes 的：
- 标题格式：`E{N}. 标题`
- EP 号与标题前缀一致
- `published_at` 是视频实际日期（非今天）
- `video_url`（YouTube 链接）已设置
- `image_url`（封面图）已设置
- `description` 非空
- 状态为 `published`

如有问题，手动修复 `published_at`：
```bash
curl -X PATCH "https://api.transistor.fm/v1/episodes/EPISODE_ID/publish" \
  -H "x-api-key: $TRANSISTOR_API_KEY" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "episode[status]=published&episode[published_at]=2026-04-05T00:00:00.000Z"
```

### 5. 排序

```bash
python3 tools/upload/reorder_episodes_by_date.py --dry-run  # 先预览
python3 tools/upload/reorder_episodes_by_date.py            # 正式更新
```

- 直接从 Transistor API + 本地 archive 取数据，不依赖外部 JSON 文件
- 按 YouTube 发布时间升序排序：E1 = 最早，E最大 = 最新
- 排序后记得检查标题前缀是否与 EP 号一致（见修复工具）

```
        ├── 标题.info.json
        └── 标题.description
```

## Transistor API 限制

**没有 DELETE 端点**。想隐藏一个 episode 只能设为 draft：
```bash
curl -X PATCH "https://api.transistor.fm/v1/episodes/EPISODE_ID/publish" \
  -H "x-api-key: YOUR_TRANSISTOR_API_KEY" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "episode[status]=draft"
```

官方 API 文档：https://developers.transistor.fm/

## 标题格式

Transistor episode 标题格式：**`E{N}. 标题`**（例：`E450. 清华教授：...`）

- 前缀：`E` + 数字 + `. `（英文句号 + 空格）
- 旧格式 `EP{N}_{标题}` 已于 2026-03-29 全部迁移

## 数据架构与权威来源

```
YouTube API
  └── tools/youtube/all_videos_full.json   ← 视频元数据权威来源（852条，含公开/私密/会员）
        用 fetch_all_videos.py 刷新（约38 units，每次全量拉取）

yt-dlp（不消耗 YouTube API 配额）
  └── archive/                              ← 媒体文件（m4a + srt + webp）
        用 tools/download/download_channel.sh 增量同步

guests.json                                ← 嘉宾-视频映射权威来源（125位嘉宾，330期）
  └── 来自 YouTube API 验证过的有效视频 ID 
  └── 推送到 GitHub 后自动驱动 lizheng.ai/guests
```

**规则**：
- 视频元数据（标题/描述/统计）→ 从 `all_videos_full.json` 读，需要最新时跑 `fetch_all_videos.py`
- 媒体文件 → 从 archive/ 读，缺失时用 yt-dlp 补
- 不把 API 数据和 archive 数据混用（archive 的 info.json 可能过时）

## YouTube API 配额说明

默认免费配额：**10,000 units/天**，足够日常使用。

| 操作 | 消耗 |
|------|------|
| `fetch_all_videos.py`（全量刷新元数据） | ~38 units |
| `apply_patches.py`（更新330个描述） | ~16,500 units，分2天 |
| 日常增量刷新（新增视频） | < 10 units |

**配额不是钱买的**：YouTube Data API 不按量计费，而是按项目申请免费额度。
如需更多，在 Google Cloud Console 提交「Quota increase」申请（免费，审核约1-2天）。
地址：`https://console.cloud.google.com/apis/api/youtube.googleapis.com/quotas?project=claude-code-youtube-493002`

## 嘉宾数据工作流（guests.json）

### 单一数据源

```
kedaibiao-channel/guests.json          ← 唯一权威来源，在这里编辑
    ↓  git push
GitHub raw URL（公开可访问）
    ↓  自动同步
lizheng.ai/guests（用户访问时实时拉取）
lizheng.ai/guests 的 SEO 预渲染（Vercel 下次构建时拉取）
```

**不要**在 lizheng-personal-site 里维护任何 guests 数据副本。

---

### 更新嘉宾数据

```bash
# 在 kedaibiao-channel 目录下
# 1. 编辑 guests.json（添加/修改嘉宾信息）
# 2. 提交并推送
git add guests.json
git commit -m "update: 嘉宾数据更新（新增 XXX）"
git push
```

推送后，`/guests` 页面的 **实时数据**（用户看到的卡片列表）立即生效——
React 在用户访问时直接从 GitHub raw URL 拉取，不依赖 Vercel 重新部署。

**SEO 预渲染**（供搜索引擎爬虫使用的静态 HTML）需要触发一次 Vercel 重新构建才刷新：
在 lizheng-personal-site 推送任意 commit，Vercel 会自动构建并用最新数据重新生成 `dist/public/guests/index.html`。

## Transistor API 已知坑

| 问题 | 原因 | 正确做法 |
|------|------|---------|
| `published_at` 总是今天 | 创建时不支持设置，PATCH /episodes/:id 会忽略 | 必须通过 `/publish` 端点：`episode[status]=published&episode[published_at]=...` |
| `image_url` 没有显示 | 创建时没传或用了 maxresdefault（404 静默失败） | 创建时传 `image_url`，用 `hqdefault.jpg`（非 maxresdefault） |
| `--upload-only-new` 路径错误 | 旧版脚本在 tools/upload/ 找 check 脚本 | 已修复：路径改为 `tools/check/check_upload_candidates.py` |
