# 网站嘉宾维护手册

这份文档面向 `lizheng.ai/guests` 的日常维护。

目标不是解释所有背景，而是让人或未来的 AI 在最短时间内知道：

- 哪些文件才是权威源
- 哪些文件不能手改
- 更新顺序是什么
- 改完以后怎么自检

## 这个功能到底包含什么

网站上的 Guest 功能有两类关键页面：

- `/guests`
  - 全部嘉宾目录页
  - 嘉宾排序主要看 `max_views`
- `/guests/:slug`
  - 单个嘉宾的可分享子页面
  - `primary_video_id` 决定精选视频
  - 下方全部访谈按 episode 的 `view_count` 排序

## Source Of Truth

### 1. 嘉宾 roster / 嘉宾与视频关系

权威文件：

- `guests.json`

这里面负责：

- 嘉宾姓名 / 公司 / 职位
- `primary_video_id`
- `all_video_ids`
- `all_urls`
- 嘉宾排序相关字段

### 2. Guest 页面视频 metadata

权威文件：

- `guest_video_metadata.json`

这里面负责：

- guest 子页的视频标题
- guest 子页的视频发布时间
- guest 子页的视频观看量

### 3. 更大的上游输入表

本地全量输入：

- `tools/youtube/all_videos_full.json`

它是更大的本地视频元数据表，但不是网站 guest 页直接依赖的 source of truth。

网站 guest 页真正依赖的是精简后的：

- `guest_video_metadata.json`

## 哪些文件不要手改

不要直接手改这些网站派生文件：

- `lizheng-personal-site/shared/guest-roster-snapshot.ts`
- `lizheng-personal-site/shared/guest-video-metadata.ts`

不要把网站 guest 页的权威数据写回网站 repo。

Guest 内容应该先改这个 repo，再同步到网站 repo。

## 推荐更新路径

### 场景 A：改嘉宾资料 / 新增或删减 guest 视频

1. 修改 `guests.json`
2. 如果 guest 引用视频有变化，同时更新 `guest_video_metadata.json`
3. 跑：

```bash
./tools/youtube/refresh_guest_page_data.sh
```

4. 提交并 push 本仓库
5. 去 `lizheng-personal-site` 跑：

```bash
pnpm refresh:guests
```

6. 提交并 push 网站仓库

### 场景 B：只更新 guest 页视频的标题 / 时间 / 播放量

1. 刷新 `tools/youtube/all_videos_full.json`
2. 跑：

```bash
python3 tools/youtube/build_guest_video_metadata.py
python3 tools/check/validate_guest_data.py
```

3. 提交并 push 本仓库
4. 去网站仓库跑：

```bash
pnpm refresh:guests
```

## 自动保护

现在已经有两层保护：

- 本地校验：
  - `python3 tools/check/validate_guest_data.py`
- GitHub Actions：
  - `Validate Guest Data`
  - 改了 `guests.json` / `guest_video_metadata.json` 后，push 或 PR 会自动跑

## 这个校验到底防什么

它会拦住这些典型错误：

- `all_urls` 更新了，但 `all_video_ids` 没更新
- `episode_count` 没同步
- `primary_video_id` 不在 `all_video_ids` 里
- `primary_url` 和 `primary_video_id` 对不上
- `guest_video_metadata.json` 缺条目
- `guest_video_metadata.json` 有重复条目
- `guest_video_metadata.json` 有已经不再被引用的陈旧条目
- guest 页需要的 `title / published_at / view_count` 为空

## 改完以后怎么验收

最少检查：

1. 目标嘉宾在 `/guests` 目录页能找到
2. 对应 `/guests/:slug` 子页能打开
3. 精选视频符合 `primary_video_id`
4. 下方列表按观看量排序
5. 小红书 / LinkedIn 链接正常
6. 标题、日期、观看量都显示正常
