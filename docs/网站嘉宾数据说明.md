# 网站嘉宾数据说明

本文只解释 `lizheng.ai/guests` 相关的数据流，不讨论播客上传、字幕校对等其他流程。

---

## 一句话结论

- **嘉宾 roster / 嘉宾-视频关系** 的唯一权威来源：`guests.json`
- **Guest 页面视频 metadata** 的权威来源：`guest_video_metadata.json`
- `tools/youtube/all_videos_full.json` 是更大的本地全量元数据表，用来生成上面的 guest metadata
- 网站线上真正直接读取的只有：`guests.json`
- 网站为了显示 episode title，额外维护了一份 **派生快照**：`lizheng-personal-site/shared/guest-video-metadata.ts`

---

## 哪些字段以 `guests.json` 为准

文件：

`kedaibiao-channel/guests.json`

它是以下信息的唯一权威来源：

- 嘉宾姓名、英文名、公司、职位
- `primary_video_id`
- `all_video_ids`
- `all_urls`
- 嘉宾排序

网站线上运行时通过 GitHub raw 直接拉这个文件：

`https://raw.githubusercontent.com/sunyuzheng/kedaibiao-content-tools/main/guests.json`

所以：

- 修改 `guests.json` 并 push 后，`/guests` 的实时数据会更新
- 不要在 `lizheng-personal-site` 里手改第二份 guest roster

---

## 哪些字段以 `guest_video_metadata.json` 为准

文件：

`guest_video_metadata.json`

它是以下信息的权威来源：

- YouTube 视频标题
- 发布时间
- view count
- 仅限 Guest 页面所需的视频级 metadata

它来自本地更大的：

`tools/youtube/all_videos_full.json`

这个大表没有通过 GitHub raw 对外发布，所以网站不直接依赖它。

---

## 网站为什么还要有一份 `guest-video-metadata.ts`

网站 repo：

`lizheng-personal-site/shared/guest-video-metadata.ts`

它不是手工编辑的 source of truth，而是部署用的**派生快照**。

存在原因：

1. `/guests/:slug` 详情页需要显示每期视频的真实标题
2. guest 子页的 SEO / Open Graph / JSON-LD 也需要这些标题和观看量
3. 线上部署需要稳定、可追踪的 guest metadata 输入
4. 网站因此把 `guest_video_metadata.json` 再快照进 repo

---

## 真实更新链路

### 场景 1：新增嘉宾，或修改嘉宾-视频关系

1. 修改 `kedaibiao-channel/guests.json`
2. 先跑 `python3 tools/check/validate_guest_data.py`
3. push `kedaibiao-content-tools`
4. 到 `lizheng-personal-site` 运行 `pnpm sync:guest-video-metadata`
5. push `lizheng-personal-site`
6. Vercel 重建后，guest 子页 SEO 和 sitemap 同步更新

### 场景 2：只更新 guest 页面视频 metadata

1. 刷新 `tools/youtube/all_videos_full.json`
2. 运行 `python3 tools/youtube/build_guest_video_metadata.py`
3. 先跑 `python3 tools/check/validate_guest_data.py`
4. 到 `lizheng-personal-site` 运行 `pnpm sync:guest-video-metadata`
5. push `lizheng-personal-site`

低频人工刷新时，可以先在内容仓库直接跑：

```bash
./tools/youtube/refresh_guest_page_data.sh
```

然后去网站仓库跑：

```bash
pnpm refresh:guests
```

---

## 不要混淆的几件事

- `guests.json` 是网站 guest roster 的权威源
- `all_urls` 和 `all_video_ids` 必须保持一一对应，提交前先跑 `validate_guest_data.py`
- `guest_video_metadata.json` 是 Guest 页面视频 metadata 的权威源
- `all_videos_full.json` 是更大的本地全量元数据表，不直接作为网站 guest 页的 source of truth
- `guest-video-metadata.ts` 是网站部署用派生文件，不是人工权威源
- `archive/**/.info.json` 可能过时，不应拿来替代 `all_videos_full.json`
